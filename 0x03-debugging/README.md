# 0x03. C - Debugging

##  Description

This project focuses on debugging C programs.
You will learn how to identify errors, fix logic issues, and understand how code behaves during execution.

---

##  Requirements

* Allowed editors: `vi`, `vim`, `emacs`
* Compilation:

  ```
  gcc -Wall -Werror -Wextra -pedantic -std=gnu89
  ```
* All files must end with a new line
* Code must follow **Betty style**
* No global variables
* Maximum 5 functions per file
* No use of `system`

---

##  Learning Objectives

By completing this project, you should be able to:

* Identify and fix bugs in C programs
* Understand compiler error messages
* Use debugging techniques (print statements, logic tracing)
* Fix infinite loops and logical errors
* Work with multiple source files and headers

---

##  Files

| File                       | Description                                              |
| -------------------------- | -------------------------------------------------------- |
| `0-main.c`                 | Tests the `positive_or_negative()` function with input 0 |
| `1-main.c`                 | Fixes an infinite loop by commenting problematic code    |
| `2-largest_number.c`       | Fixes logic to correctly find largest of three numbers   |
| `3-print_remaining_days.c` | Fixes leap year handling and date calculations           |
| `main.h`                   | Contains all function prototypes                         |

---

##  Compilation Example

```bash
gcc -Wall -Werror -Wextra -pedantic -std=gnu89 file1.c file2.c -o output
```

---

## ▶️ Usage

Run the compiled program:

```bash
./output
```

---

##  Debugging Concepts Covered

* Infinite loops
* Conditional logic errors
* Function prototypes and header files
* Multi-file compilation
* Leap year calculations
* Edge case handling
